# Data Directory Guide

This directory contains external baselines, cross-run summaries, and a
review-ready organized copy of the computed analysis outputs.

## Recommended Entry Point

Start with:

`organized_analysis_outputs/README.md`

That folder is the clean, review-facing package. It keeps the original project
outputs intact, but copies the key CSV/JSON/MD files into functional
subfolders with clearer names.

## Top-Level Folders

### `organized_analysis_outputs/`

Curated analysis package for paper writing, review response, and result lookup.

It contains:

- `01_human_reference_alignment/`: how often each judge ranks the human
  reference translation first.
- `02_judge_agreement_and_consensus/`: pairwise agreement, Borda consensus, and
  model-vs-consensus summaries.
- `03_self_excluded_judge_sets/`: preferred sensitivity analyses excluding the
  candidate generator itself.
- `04_cluster_robust_regressions/`: cluster-robust pairwise logistic regression
  outputs grouped by feature source.
- `05_feature_matrices/`: per-candidate feature matrices used as regression
  inputs.
- `06_stability_and_review_summaries/`: cross-version stability tiers and
  concise finding notes.
- `07_native_baseline_syntax/`: native financial-text baseline syntax
  distributions and candidate-vs-native comparisons.
- `MANIFEST.csv`: trace each organized file back to its original source path.

Naming has been normalized inside this package:

- historical `five_models` -> `all_judges_5model`
- historical `four_models` -> `legacy_four_judges_not_self_excluded`
- actual self-exclusion -> `judges_excluding_candidate_generator_<model_slug>`

### `analysis_summaries/`

Cross-run summary files generated by analysis scripts. These include the
self-excluded stability tier tables and concise finding notes. The organized
package copies these files into `organized_analysis_outputs/06_stability_and_review_summaries/`.

### `external_baselines/`

External native financial text baselines and derived Stanza syntax summaries.
These support target-language/native-text comparisons and translationese-related
interpretation. The organized package copies the main Stanza summaries into
`organized_analysis_outputs/07_native_baseline_syntax/`.

## Interpretation Notes

- Prefer cluster-robust regression files for inference.
- Treat accuracy fields as in-sample explanatory fit, not held-out prediction
  accuracy.
- Use self-excluded judge-set outputs when discussing self-preference.
- Use translationese-subfeature outputs for the main translationese claim; keep
  the old composite `translationese_score` as historical or supplementary
  evidence.
