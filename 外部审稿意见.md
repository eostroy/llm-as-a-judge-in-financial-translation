## 二、方法学问题（Major）

### 2.1 自偏好（Self-Preference）混淆未完全排除

这是实验设计中的核心问题。两个版本的候选译文来源如下：

| 版本 | 候选生成模型 | 评审模型中含同族模型？ |
|------|-------------|----------------------|
| kimi-version | Kimi K2.5 | 是（Kimi K2.5 同时是评审模型） |
| claude-version | Claude Sonnet 4.6 | 是（Claude Sonnet 4.6 同时是评审模型） |

两个版本都包含生成候选的模型作为评审者之一。虽然"用另一版本做对照"的设计方向是对的，但更干净的对照应该是交叉设计或排除自身评审后的四模型分析。项目中已经生成了 `four_models.*` 的分析文件，说明作者意识到了这个问题。

**建议**：明确披露这一 confound，在正文中将 five_models 和 four_models 的结果都呈现给读者，并报告排除与候选生成器相同的评审模型后的敏感性分析结果。

### 2.2 自定义梯度下降替代标准库（Robustness 风险）

所有 logistic 回归均使用**手写梯度下降**实现，而未使用 sklearn、statsmodels 等标准库。主要风险：

**(a) 无法确保收敛**：固定 epoch 数（2400–3500）没有收敛判据，也未检查 gradient norm 或 loss change。虽然 loss 曲线大概率已经平坦，但没有形式化验证。

**(b) 超参数不一致**：

| 分析脚本 | 学习率 | Epochs | L2 正则化 |
|---------|--------|--------|-----------|
| analyze_ec_deep_features | 0.07 | 2400 | 0.015 |
| analyze_local_embedding_features | 0.08 | 2500 | 0.01 |
| analyze_crosslingual_nli_features | 0.08 | 2500 | 0.01 |
| analyze_joint_pairwise_features | 0.07 | 3500 | 0.01 |
| analyze_rankings_pairwise_features | 0.08 | 2500 | 0.01 |

不同 L2 惩罚强度意味着不同特征族之间的系数不可直接比较。虽然后来做了联合回归统一参数（3500 epochs, L2=0.01），但各特征族的单独分析报告的是不同正则化强度下的结果。

**(c) 自定义矩阵求逆**：`analyze_cluster_robust_pairwise_feature_families.py:83-109` 在 cluster-robust 协方差计算中使用纯 Python 手写矩阵求逆，而非 numpy.linalg.inv 或 scipy，引入了不必要的数值稳定性风险。

**建议**：用 sklearn.linear_model.LogisticRegression 验证手写实现的系数一致性，至少在附录中报告验证结果。

### 2.3 无多重比较校正

联合回归含 14 个特征 × 5 个模型 = 70 个检验项（每条件）。所有报告的 p 值均基于 Wald 检验的名义显著性水平，未做任何多重比较校正（Bonferroni、Benjamini-Hochberg 等）。以 α=0.05 和 70 次独立检验计算，期望约有 3.5 个假阳性。

**建议**：论文中基于 "p < 0.05" 的显著性声明应标注为"未经多重比较校正的探索性分析"，或在讨论中使用更保守的显著性阈值（如 p < 0.01）。

### 2.4 无训练/测试分离

所有 logistic 回归报告的 `training_accuracy` 都是**样本内拟合精度**（in-sample），不是预测精度。联合回归的 in-sample accuracy 在 0.63–0.73 之间（不同模型），说明即使 14 个特征全部使用，模型对 pairwise 偏好的拟合也较为有限。这本身是一个有意义的发现——评审模型的偏好受许多未测量因素影响——但论文应明确说明这是 in-sample fit，不构成预测模型的性能声明。

**建议**：将 "training_accuracy" 改为 "in-sample accuracy"，在方法部分明确说明无 train/test split，并在讨论中指出模型拟合度有限所暗示的未测量偏好因素。

### 2.5 NLI 特征的多重模型依赖

`nli_boundary_risk`、`nli_omission_risk`、`nli_contradiction_risk` 和 `bidirectional_entailment` 均来自 mDeBERTa-v3-base-mnli-xnli 模型的 softmax 概率，而非人工标注。这意味着：

- NLI 特征本身就带有模型误差
- joint regression 中实际分析的是"DeBERTa 判断的语义关系"与"LLM 评审偏好"的关系
- 如果 DeBERTa 系统性地误判某些类型的蕴含关系（如财经领域的隐含因果关系），这些偏差会传导到回归结果中

这虽然不是致命问题（很多 NLP 研究都这样做），但论文中应明确讨论这一依赖链对解读的限制。

### 2.6 聚类稳健标准误的系数复用

`analyze_cluster_robust_joint_logistic.py` 和 `analyze_cluster_robust_pairwise_feature_families.py` 中的聚类稳健标准误计算**重用了来自原始未聚类拟合的系数**，仅对协方差矩阵做了聚类修正。虽然系数本身是一致估计量（MLE 的一致性不依赖于误差结构），但严格来说更严谨的做法是在估计阶段就纳入聚类结构（如 GEE 或 mixed-effects logistic regression）。

正面来看，有限样本修正因子 `(g/(g-1)) * ((n-1)/(n-k))` 的使用是正确的，200 个聚类、每簇 3 个观察值的设置也是合理的。

## 三、数据分析问题（Moderate）

### 3.1 `sentence_compression_ratio` 跨版本方向反转

这个变量在 EC 方向的系数方向在 kimi-version 和 claude-version 之间**完全反转**：

- kimi-version EC：所有 5 个模型的系数均为**负**（评审模型偏好更紧凑 / 长度比更小的译文）
- claude-version EC：显著模型中 3/5 系数为**正**（评审模型偏好更展开 / 长度比更大的译文）

这一反转难以在"模型偏好译文长度"的单维度框架内解释，更可能反映了 kimi-version 和 claude-version 生成的候选译文在该维度上的分布差异。论文在讨论该特征时需明确强调版本依赖性，不能将其报告为跨版本稳健的发现。

### 3.2 `translationese_score` 的构造权重缺乏理论依据

```python
raw = 0.35 * min(marker_hits / 12, 1.5) + 0.25 * min(de_ratio / 0.08, 1.5)
raw += 0.20 * long_sentence_ratio + 0.20 * min(passive_or_nominal / 5, 1.5)
```

权重 0.35、0.25、0.20、0.20 和截断阈值 12、0.08、5 是任意设定的，缺乏理论依据或经验校准。虽然这在探索性分析中可接受，但论文应说明这些是启发式构造，其系数数值（而非方向）不应被强解释。好消息是 translationese_score 在 EC 方向所有模型和版本中一致性极强（10/10 全部 p < 0.001），这减轻了权重任意性的担忧——信号的稳健性不依赖于精确的权重设定。此外，项目新增的 Stanza 母语基准句法比较（`analyze_stanza_native_baseline_syntax.py`）通过将候选译文的句法特征与原生中文/英文财经文本对比，提供了更经验化的翻译腔评估路径，可以作为 heuristic translationese_score 的有益补充。

### 3.3 目标语语言模型较小

- EC 方向使用 `uer/gpt2-chinese-cluecorpussmall`（小型中文 GPT-2）
- CE 方向使用 `distilgpt2`（小型英文 GPT-2）

这些模型对财经领域专业文本的自然度评估能力有限。虽然它们是合理的目标语流畅度 proxy，但对其系数的解读应以"小型目标语语言模型判断的自然度"而非"目标语自然度"为准确表述。

### 3.4 EC 方向 LaBSE 语义相似度不显著是一个应被正面强调的重要发现

`crosslingual_embedding_similarity` 在 EC 方向的显著性：

- kimi-version EC：仅 1/5 模型显著
- claude-version EC：0/5 模型显著

这与 CE 方向（10/10 全部显著）形成鲜明对比。**这是一个有价值的跨方向稳健发现**：英文到中文的翻译质量评估中，简单的跨语言语义向量相似度不是评审模型的核心判断依据，评审模型更依赖翻译腔、数字控制和结构组织等目标语端特征。这一发现在论文中应被正面强调，而非一笔带过。

### 3.5 缺失参考译文的上界分析

三个候选译文中有一个是人工参考译文（`human_translation`）。但分析中似乎没有系统地报告各评审模型在盲评条件下**选人工译文为最佳（rank=1）的频率**。这一指标对于评估 LLM-as-a-judge 的 "alignment with human preference" 至关重要。

例如，如果某评审模型选人工译文为 top-1 的比例不足 50%（随机基线约 33%），则说明该模型的质量标准与人类存在系统偏差。所有模型在 blind condition 下的 top-1 中人工译文占比应作为一个核心描述性统计量报告。

### 3.6 CE 方向遗漏风险显著性的版本差异

`nli_omission_risk` 在 CE 方向的显著性呈现版本差异：

- kimi-version CE：仅 1/5 模型显著
- claude-version CE：4/5 模型显著

论文中说"受到源文信息完整性与遗漏风险影响"需要区分版本讨论，不宜作为 CE 方向的统一结论。建议表述为"遗漏风险在 claude-version CE 方向对评审偏好有稳定解释力，在 kimi-version CE 方向则不够稳健"。

## 四、解读与表述问题（Minor）

### 4.1 "较高共识"缺乏参照基准

论文称"EC 方向平均一致性约为 0.835"表征了"较高共识"，但未给出判断基准。随机基线为 0.5（两个模型随机选 winner），所以 0.835 确实远高于随机。但建议提供更有信息量的参照：

- 人类翻译质量评估者之间的 inter-annotator agreement 通常在 0.7–0.85 之间（取决于任务和评估维度）
- 如果提供这一参照，"模型间一致性接近或略高于人类标注者之间的一致性"将是一个信息量更丰富的陈述

注：项目新增的母语基准句法分析（`analyze_stanza_native_baseline_syntax.py`）已为句法特征提供了原生文本分布作为参照（z-score），这是正确的做法。若能对一致性指标也提供类似的基准参照将使论证更完整。

### 4.2 共识方法未讨论灵敏度

共识排序使用 Borda count（将每个模型的排名转化为 Borda 分数后求和排序）。Borda 方法的已知弱点是易受候选集合组成影响（不满足 independence of irrelevant alternatives）。对于三候选场景这可能不是主要问题，但应在方法部分简要提及并讨论其对结论的潜在影响。

### 4.3 four_models vs five_models 的选择逻辑不透明

项目中同时存在 `four_models.*` 和 `five_models.*` 的分析文件。从文件名无法直接推断 four_models 排除了哪个评审模型，以及排除标准是什么。论文应说明排除逻辑，并在附录中报告 four_models 分析与 five_models 分析的异同。

### 4.4 术语精确性

部分变量命名与数据实际含义不符，项目文档中已明确承认：

| 历史命名 | 实际含义 | 论文建议用词 |
|---------|---------|------------|
| `sentence_compression_ratio` | 目标语/源语长度比 | target/source length ratio / translation expansion ratio |
| `deep_features` | 规则式 proxy 特征 | rule-based proxy features |
| `local_embedding` | LaBSE 跨语言语义相似度 | LaBSE cross-lingual embedding similarity |

论文中应使用这些更精确的术语。

### 4.5 LaBSE 所用模型确认

`analyze_local_embedding_features.py` 的默认模型为 `bert-base-multilingual-cased`（第 21 行），但项目文档中多处提到使用的是 LaBSE。虽然 cache 文件路径中包含 `LaBSE` 字样，暗示实际运行时使用了 LaBSE 模型（通过 `--model` 参数覆盖了默认值），但脚本的默认值不一致。论文和 method note 应明确确认实际使用的模型名称和版本。

## 五、研究设计中的亮点

在列举问题的同时，以下几方面值得肯定：

1. **双版本交叉验证设计**：用 kimi-version 和 claude-version 对照检查候选来源效应，是严谨的实验设计思路。
2. **盲评实现规范**：用固定 seed 的随机打乱 + `candidate_position_map` 记录原始标签，流程可追溯。
3. **多层特征分析框架**：从一致性 → 语义 → 句法 → 控制变量 → 联合回归的递进分析逻辑清晰。
4. **聚类稳健标准误的事后修正**：意识到 3 个 pairwise 观察值的非独立性并用 cluster-robust SE 修正，是负责任的统计实践。README 也已明确引导读者优先使用 cluster_robust 文件，降低了误用未修正结果的风险。
5. **母语基准句法对比**（新增）：`analyze_stanza_native_baseline_syntax.py` 下载了原生中文/英文财经语料（Kenpache），计算候选译文句法特征相对于原生文本的 z-score 偏离度，为"翻译译文在多大程度上偏离目标语规范"提供了经验参照。这比单纯依赖启发式 translationese_score 更可靠。
6. **项目文档完备**：`FILE_INDEX.csv`、`指标词典.md`、`PROJECT_STRUCTURE_AND_DATA_GUIDE.md` 等导航文档齐全，极大提高了可复现性。项目从平铺结构重构为两版本平行布局后，导航更加清晰。
5. **项目文档完备**：`FILE_INDEX.csv`、`指标词典.md`、`PROJECT_STRUCTURE_AND_DATA_GUIDE.md` 等导航文档齐全，极大提高了可复现性。
7. **发现的方向性差异价值高**：EC 和 CE 评审机制的本质差异（语义对应 vs 目标语特征）是核心贡献。
