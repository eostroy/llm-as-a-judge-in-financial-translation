# 项目结构与数据指南

这个文件是项目的“导航层”。它不替代原始数据，而是帮助你快速判断每个目录、每类文件和主要文件名代表什么。

## 一、版本目录

- `kimi-version/`：国产模型生成候选译文版本。用于观察在国产模型候选下，五个评审模型的排序偏好。
- `claude-version/`：海外模型生成候选译文版本。用于检查候选生成来源是否导致模型自偏好。

两个版本的内部结构基本一致，因此可以用同一相对路径比较两版结果。

## 二、翻译方向

- `ec/`：English-to-Chinese，英译中。语料来自 FFN 财经新闻子集。
- `ce/`：Chinese-to-English，中译英。语料来自 ECPCFE 财经/经济语料子集。

每个方向包含 200 条样本，每条样本有三个候选译文，并由多个 LLM 进行盲评排序。

## 三、主要目录含义

- `datasets/`：任务数据。最常用的是 `*.with_candidates.shuffled.json`，这是候选位置打乱后的盲评输入。
- `results/model_based_metrics/rankings/json/`：各评审模型输出的原始排序结果。
- `results/model_based_metrics/analysis/pilot/`：模型一致性、共识排序、LaBSE、NLI、目标语语言模型、联合回归等结果。
- `results/parser_derived_syntactic_metrics/analysis/pilot/`：依存句法分析结果，包括依存深度、平均依存距离、归一化依存距离等。
- `results/rule_based_proxy_features/analysis/pilot/`：规则/经验式控制特征，包括数字、实体、语域、translationese 等代理指标。
- `results/*/analysis/human_reference/`：模型首选译文与人类/参考译文的比较。
- `logs/`：生成、排序和分析过程的日志。

## 四、常用文件名怎么看

| 文件名片段 | 数据意义 | 主要用途 |
| --- | --- | --- |
| `with_candidates.shuffled.json` | 候选 A/B/C 已打乱的盲评数据 | 排序脚本输入 |
| `ranked.<model>.json` | 某个评审模型给出的 A/B/C 排序 | 原始 judge 结果 |
| `five_models.pairwise_agreement.csv` | 五个模型两两一致率 | 支撑“整体一致性” |
| `five_models.model_vs_consensus_agreement.csv` | 单个模型与共识排序的一致性 | 比较模型间差异 |
| `five_models.consensus_borda_summary.csv` | Borda 共识排序摘要 | 查看群体首选候选 |
| `local_embedding_features.*` | LaBSE 跨语言语义相似度 | 分析语义贴合度 |
| `crosslingual_nli_features.*` | 跨语言 NLI 特征 | 分析蕴含、遗漏、矛盾风险 |
| `syntax_info_features.*` | 依存句法特征 | 分析句法结构与依存跨度 |
| `deep_features.*` | 规则/经验式控制特征 | 辅助控制，不宜单独强解释 |
| `target_lm_features.*` | 目标语语言模型自然度 | 辅助 fluency/naturalness proxy |
| `joint_features.pairwise_observations.json` | 回归用 pairwise 展开表 | 每样本 3 个 pairwise 判断 |
| `joint_features.pairwise_logistic_preferences.csv` | 原始联合 logistic 回归 | 看系数方向 |
| `joint_features.cluster_robust_pairwise_logistic.csv` | 按样本 ID 聚类稳健标准误后的联合回归 | 当前推荐用于统计推断 |

## 五、需要特别注意的误导性名称

### `sentence_compression_ratio`

这个名称容易误导。它不是通用意义上的“压缩质量”指标，而是目标语和源语之间的长度比。

- EC：中文译文字数 / 英文源文词数。
- CE：英文译文词数 / 中文源文字数。

建议在解释时把它理解为 `target_source_length_ratio` 或“译文展开程度/目标源长度比”。尤其在 EC 中，数值更高可能意味着中文译文更充分、更显性地承载信息，而不一定意味着结构负担更重。

### `deep_features`

这个名字不是“深度模型特征”的意思，而是历史命名。当前应理解为规则/经验式 proxy 特征，包括数字、实体、语域和 translationese 风格代理指标。

### `local_embedding`

当前主要指本地 LaBSE 模型算出的跨语言语义相似度，不是所有 embedding 模型的统称。

## 六、当前解释时的优先级

1. 先看模型一致性和共识排序，确认排序行为是否稳定。
2. 再看 LaBSE 和 NLI，解释语义贴合度、遗漏风险和蕴含稳定性。
3. 再看依存句法，解释结构负担、依存跨度和局部结构控制。
4. 最后看规则/经验式 proxy 和目标语 LM，作为辅助控制。
5. 做统计解释时，优先使用 `cluster_robust` 联合回归文件。
